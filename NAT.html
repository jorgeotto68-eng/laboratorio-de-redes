Explorando `tracert` y `NAT`

¿Qué es `tracert`?

Es una herramienta de diagnóstico que mapea la ruta que un paquete de datos toma para llegar a un destino.

tracert google.com

Muestra cada "salto" (router) y el tiempo de respuesta.

¿Cómo Funciona `tracert`?

Envía paquetes con un valor de TTL (Time-to-Live) que aumenta progresivamente. Cada router lo decrementa en 1.

  • TTL=1, el primer router lo descarta y avisa.
  • TTL=2, el segundo router lo descarta y avisa.
  • Y así sucesivamente, hasta que el paquete llega al destino.

¿Qué es `NAT`?

**Network Address Translation** es la técnica para que múltiples dispositivos con IP privada compartan una única IP pública para acceder a Internet.

¿Por qué es importante `NAT`?

Principalmente para **conservar direcciones IPv4**. Permite que miles de dispositivos se conecten a Internet a través de una sola dirección pública, superando el límite de direcciones del protocolo.

Tipos de `NAT`

  • **NAT Estático:** Un mapeo uno a uno.
  • **NAT Dinámico:** Un grupo de IPs privadas se mapea a un grupo de IPs públicas.
  • **PAT (Port Address Translation):** El más común. Usa puertos para mapear múltiples IPs privadas a una sola IP pública.
